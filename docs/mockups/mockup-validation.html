Contextual Validation

Errors caught inline where they happen — not as silent log warnings or broken behavior

Before: errors were invisible
What happened before this PR
Setting min_brightness: 80 and max_brightness: 20 was silently accepted. Enabling detect_non_ha_changes without take_over_control was silently accepted. Setting min_sunrise_time after max_sunrise_time was silently accepted.

All produced confusing runtime behavior with no indication of what went wrong. Users spent hours debugging in GitHub issues.
After: errors caught at the point of configuration
Step 1 Brightness range inverted
User sets min brightness higher than max brightness.
min_brightness
Lowest brightness (%) applied at night.
80
Minimum brightness must be less than or equal to maximum brightness
max_brightness
Highest brightness (%) applied during the day.
20
Error appears immediately on Submit. User fixes it before it causes confusing runtime behavior.
Step 1 Color temp range inverted
User sets warmest color temp higher than coldest.
min_color_temp
Warmest color temperature (Kelvin) applied at night.
6500
Minimum color temperature must be less than or equal to maximum color temperature
max_color_temp
Coldest color temperature (Kelvin) applied around midday.
2700
Step 4 Missing dependency: take_over_control
User enables detect_non_ha_changes but take_over_control is off.
take_over_control
Pauses adaptation when another source changes a light.
Off
depends on
detect_non_ha_changes
Detects changes made outside Home Assistant by polling lights.
On
This option requires take_over_control to be enabled
adapt_only_on_bare_turn_on
Skips adaptation when a light is turned on with explicit color or brightness.
On
This option requires take_over_control to be enabled
Both dependent options show errors simultaneously. User sees exactly which toggle they need to flip.
Step 3 Sunrise time range inverted
User sets min sunrise after max sunrise — previously caused silent misbehavior.
min_sunrise_time
Earliest allowed sunrise time (HH:MM:SS).
09:00:00
Minimum sunrise time must be earlier than maximum sunrise time
max_sunrise_time
Latest allowed sunrise time (HH:MM:SS).
07:30:00
sunrise_offset
Shifts the sunrise time (seconds).
0
Caught at config time instead of causing the effective sunrise to silently default to an unexpected value.
